\section{The Missing Middle}

Agent frameworks decide what should happen next. Gateways route model requests. Workflow engines preserve durable state. Provider APIs expose model-specific capabilities. Tool protocols expose external systems. Observability platforms record behavior. Evaluation systems measure outcomes.

Each layer owns part of execution. Few layers can optimize across the whole execution graph. A single agentic workflow may require decisions across all of the following:

\begin{itemize}[leftmargin=1.5em]
  \item model selection and provider routing;
  \item retry, fallback, and timeout policy;
  \item retrieval ordering and context assembly;
  \item duplicate context elimination and prompt-cache layout;
  \item tool-call parallelization and side-effect control;
  \item validation, regression testing, and human approval;
  \item cost accounting, latency budgeting, and audit replay;
  \item provider allowlists, data-retention rules, and trace redaction.
\end{itemize}

These decisions interact. A cache decision may depend on context layout. A routing decision may depend on privacy class. A retry policy may depend on idempotency. A validation policy may change whether lossy compression is acceptable. Local optimization is therefore insufficient.

\begin{claim}[Execution graph as optimization boundary]
The unit of optimization in agentic systems should not be a single prompt or model call. It should be the full model/tool/retrieval/context execution graph.
\end{claim}
